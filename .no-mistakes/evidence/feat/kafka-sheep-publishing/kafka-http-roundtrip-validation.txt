End-to-end HTTP -> Kafka round-trip (validation run, 2026-07-01)
================================================================

Goal (user intent): accepted usage events ("sheeps") published into a Kafka
topic via a producer wired into the record_accepted seam, using a flattened
CloudEvents wire format, subject as the partition key, non-blocking delivery.

Setup
-----
Broker : docker-compose.kafka.yml  (apache/kafka 3.9.0, KRaft) on localhost:9092
Server : ./target/debug/server  with  KAFKA_BROKERS=localhost:9092
                                       KAFKA_TOPIC=sheeps-http-evidence  PORT=8099
Flow   : HTTP POST /api/v1/sheeps -> KafkaProducer -> topic "sheeps-http-evidence"
         -> consumed with kafka-console-consumer (--from-beginning)

Design decisions confirmed by the live data
-------------------------------------------
  * subject is the Kafka partition key          -> keys: customer-42 / customer-7 / customer-99
  * flattened CloudEvents payload               -> id / type / source / subject / time(unix s) / data
  * specversion + received_at carried as headers-> headers above
  * occurred_at (payload "time") is distinct from received_at (header):
      customer-42 payload time 1781944200 == 2026-06-20T08:30:00Z (the input's explicit time)
      received_at header 1782920787          == 2026-07-01T15:46:27Z (ingestion "now")
  * missing time defaults to ingestion now      -> customer-7 has no input time; its payload
                                                   time == received_at (1782920787)
  * one clock read per batch                    -> customer-7 and customer-99 share received_at 1782920787
  * batch validation is all-or-nothing          -> the invalid batch (subject "customer-should-not-appear")
                                                   produced 0 records; only the 3 valid events landed
